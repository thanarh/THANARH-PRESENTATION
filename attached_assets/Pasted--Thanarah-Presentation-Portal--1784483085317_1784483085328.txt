# Thanarah Presentation Portal

بوابة عرض شديدة الخصوصية لعرض رؤية المشروع، المعمارية، مراحل التنفيذ، نموذج الأعمال، الواجهات التخيلية والخطط المالية على المؤسسين والمستثمرين والشركاء، من خلال دعوات مؤقتة وصلاحيات دقيقة وتسجيل كامل لكل نشاط.

هناك نقطة أمنية مهمة قبل البدء: **لا توجد تقنية ويب تستطيع منع تصوير الشاشة بكاميرا هاتف آخر أو جعل المحتوى واضحًا للعين ومشوّهًا للكاميرات بشكل مضمون**. يمكن إضافة تأثيرات بصرية مقاومة للتصوير، وعلامات مائية ديناميكية، وتتبع التسريب، لكن الادعاء بأن التصوير الخارجي سيصبح مستحيلًا سيكون غير صحيح. الحل الأقوى هو جعل كل نسخة ظاهرة مرتبطة بهوية صاحبها، حتى يمكن معرفة مصدر أي تسريب.

الأمان ومنع التسريب:

مهم:
لا تدّعِ أن المتصفح يستطيع منع Screenshot أو تصوير الشاشة بكاميرا خارجية بشكل كامل.

طبق نظام حماية واقعي متعدد الطبقات:

1. Dynamic Personalized Watermark
يظهر فوق المحتوى بشكل خفيف ومتكرر ويحتوي على:
- اسم المستخدم.
- البريد المختصر.
- رقم الجلسة.
- التاريخ والوقت.
- عبارة Confidential.
- كود تتبع غير واضح للمستخدم.

تتغير مواضع العلامة المائية ببطء لمنع إزالتها بسهولة.

2. Forensic Watermark
أنشئ نمطًا بصريًا خفيفًا مختلفًا لكل جلسة، يسمح بتحديد الجلسة المحتمل أن تكون مصدر الصورة المسربة.

3. Visible Watermark
تظهر بطريقة لا تعيق القراءة، لكنها موجودة في كل شاشة.

4. Session-Specific Rendering
كل مستخدم يرى نسخة مرتبطة بمعرف جلسة.

5. Screenshot Detection
على الويب لا يمكن ضمان اكتشاف Screenshot.
يمكن تسجيل مؤشرات فقط مثل:
- فقدان تركيز الصفحة.
- تغيّر Visibility.
- استخدام Print Screen في بعض البيئات.
- فتح أدوات الطباعة.
- محاولات النسخ المتكررة.

لا تعتبر هذه الأدلة مؤكدة وحدها.

6. Mobile Native Protection مستقبلًا
في تطبيقات الهاتف الأصلية:
- Android FLAG_SECURE.
- iOS Screen Capture Detection.
لكن هذا لا يمنع تصوير الجهاز بكاميرا خارجية.

7. Screen Recording Warning
عند اكتشاف مشاركة شاشة أو تسجيل في بيئات تدعم ذلك:
- إظهار تحذير.
- تسجيل Security Event.
- إمكانية إخفاء الأقسام الحساسة حسب السياسة.

8. External Camera Resistance
يمكن إضافة:
- Texture خفيف.
- نمط Moiré دقيق.
- Micro-pattern متحرك.
- Temporal dithering خفيف.
لكن يجب أن يكون اختياريًا؛ لأنه قد يؤثر على العين والشاشات، ولا يقدم حماية مضمونة ضد الكاميرات.

لا تجعل النص الطبيعي مشوهًا للمستخدم.

9. Disable Common Leakage Actions
- تعطيل النقر الأيمن في صفحات العرض.
- تعطيل النسخ للنصوص الحساسة.
- تعطيل السحب والإفلات.
- تعطيل الطباعة.
- منع Save Page.
- منع تحديد النص في العرض، مع إبقاء إمكانية الوصول Accessibility في الاعتبار.
- منع فتح الصور الحساسة في تبويب مباشر.
- استخدام Signed URLs.
- منع Hotlinking.

10. DevTools
لا تدّعِ إمكانية منع DevTools بالكامل.
يمكن:
- عدم إرسال بيانات غير مصرح بها أصلًا.
- تطبيق الصلاحيات في الخادم.
- عدم الاعتماد على إخفاء عناصر الواجهة.
- عدم وضع الأسرار في Frontend.
- تعطيل Source Maps في Production أو حمايتها.
- Minification.
- Content Security Policy.
- Trusted Types إن أمكن.
- منع Inline Scripts غير الضرورية.

11. Rate Limiting
طبق Rate Limits على:
- تسجيل الدخول.
- التحقق.
- الدعوات.
- إعادة إرسال البريد.
- استعادة كلمة المرور.
- تحميل المحتوى.
- API الحساسة.

12. Brute Force Protection
- زيادة تأخير تدريجية.
- Captcha بعد عدد من المحاولات.
- قفل مؤقت.
- إشعار صاحب الحساب.
- تسجيل الحدث.

13. Device and Session Controls
- عرض جميع الجلسات.
- تسجيل خروج من جهاز محدد.
- تسجيل خروج من جميع الأجهزة.
- منع أكثر من جهاز حسب الدعوة.
- انتهاء جلسة تلقائي.
- Idle Timeout.
- Absolute Session Timeout.
- تجديد آمن للـTokens.
- Refresh Token Rotation.

14. Cookies
- HttpOnly.
- Secure.
- SameSite Strict أو Lax حسب التدفق.
- CSRF Protection.
- عدم تخزين Tokens الحساسة في Local Storage.

15. Passwords
- Argon2id أو bcrypt بإعداد قوي.
- Password Policy.
- منع كلمات المرور المسربة عند توفر خدمة مناسبة.
- عدم إرسال كلمة المرور بالبريد.
- تغيير إجباري عند الاشتباه.

16. Encryption

- تشفير البيانات الحساسة في قاعدة البيانات.
- تشفير النسخ الاحتياطية.
- فصل مفاتيح التشفير عن قاعدة البيانات.
